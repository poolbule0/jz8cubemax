# 打包说明

## 快速打包

### 方法一：使用批处理文件（推荐）

1. 双击运行 `build.bat`
2. 等待打包完成
3. 生成的exe文件位于 `dist\JZ8代码生成工具.exe`

### 方法二：手动打包

1. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```

2. 运行打包脚本：
   ```bash
   python build_exe.py
   ```

3. 生成的exe文件位于 `dist\JZ8代码生成工具.exe`

## 打包特性

- ? **无控制台窗口**：使用 `--windowed` 和 `--noconsole` 选项，运行时不会显示黑框
- ? **单文件打包**：使用 `--onefile` 选项，所有依赖打包到一个exe文件
- ? **包含数据文件**：自动包含 `数据手册` 和 `示例项目` 目录
- ? **完整模块支持**：包含所有必要的Python模块和tkinter组件

## 注意事项

1. **首次打包**：首次打包可能需要较长时间（5-10分钟），因为需要收集所有依赖
2. **文件大小**：打包后的exe文件可能较大（50-100MB），这是正常的，因为包含了Python解释器和所有依赖
3. **杀毒软件**：某些杀毒软件可能会误报，这是PyInstaller打包的常见情况，可以添加信任
4. **运行环境**：打包后的exe可以在没有安装Python的Windows系统上运行

## 故障排除

### 如果打包失败

1. 确保已安装Python 3.7+
2. 确保已安装所有依赖：`pip install -r requirements.txt`
3. 检查是否有文件被占用
4. 查看错误信息，根据提示修复

### 如果exe运行失败

1. 检查是否缺少数据文件（数据手册、示例项目）
2. 尝试在命令行运行exe查看错误信息
3. 确保Windows系统支持tkinter（通常已内置）

## 自定义选项

如果需要修改打包选项，可以编辑 `build_exe.py` 文件：

- 修改exe文件名：更改 `--name` 参数
- 添加图标：将 `--icon=NONE` 改为 `--icon=图标文件路径.ico`
- 修改输出目录：添加 `--distpath=输出目录`
